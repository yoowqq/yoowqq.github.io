Vue面经

1. Vue的生命周期

核心概念：vue生命周期是组件从创建、挂载、更新到销毁的完整过程，每个阶段都提供了钩子函数。

生命周期钩子函数（以vue2为例）：

  1. 创建阶段

    • beforeCreate：

      触发时机：实例初始化后，在数据观测（data observer）和事件监听前
      特征：无法访问data，props，method
      用途：插件注入

    • created：

      触发时机：数据观测（data observer）和事件监听后，尚未挂载DOM前

      特征：可访问data，props，method

      用途：发起异步请求（减少白屏事件）

  2. 挂载阶段

    • beforeMount：

      触发时机：模板已编译为渲染函数，虚拟DOM已创建但未渲染为真实DOM

      特征：无法访问真实 DOM（this.$el 仍然是占位符），可访问 this.$options.render（查看编译后的渲染函数）

    • mounted：

      触发时机：实例已挂载到真实 DOM

      特征：可操作真实DOM，可通过$refs访问子组件（但子组件不一定全部挂载完（尤其异步和条件渲染的）

      用途：处理一些依赖DOM的操作

  3. 更新阶段

    • beforeUpdate：

      触发时机：数据变化后，虚拟DOM重新渲染前

      特征：数据已更新但视图未同步

      用途：记录视图更新前的信息（表单编辑前的快照）

    • updated：

      触发时机：虚拟DOM重新渲染并打补丁到真实DOM后

      特征：DOM已更新为最新状态

  4. 销毁阶段

    • beforeDestory：

      触发时机：实例销毁前(实例仍完全可用)

      特征：所有功能(data/methods/DOM)仍可用

      用途：清除定时器/事件监听器,取消未完成的网络请求

    • destroyed：

      触发时机：实例销毁后(所有子实例也被销毁)

vue3在生命周期上的改动：

  • 移除beforeCreate和created，改成在setup（）里面编写逻辑
  • 新增onRenderTracked：追踪响应式依赖时触发（开发调试用）
  • 新增onRenderTriggered：响应式依赖触发重新渲染时调用
  • beforeDestory和destroyed改为beforeUnmount和unmounted
  • 钩子函数以onXXX形式导入，在setup（）里面使用

2. Vue的响应式原理

Vue2使用Object.defineProperty对对象的每个属性进行劫持，转换为getter/setter，每一个组件实例对应一个watcher，通过“依赖收集”机制通知更新视图。缺陷：新增属性和通过数组索引修改无法响应。

Vue3改用proxy作为对象的代理，可以拦截和监听对象和数组的变化；通过track函数收集依赖，trigger函数触发更新。使用reflect提供了与proxy拦截器方法一一对应的方法，可以方便地执行默认行为。

3. 依赖机制

Vue2：数据读取时触发getter，将当前watcher加入到Dep(依赖集合)；数据修改时触发setter，Dep通知所有关联的watcher执行更新，watch通知组件执行回调函数等。

Vue3：Vue 会维护一个全局的 targetMap（基础类型使用set），它是一个 WeakMap，用于存储每个响应式对象及其属性的依赖关系。每个属性会对应一个 Dep（依赖集合），里面存放了所有依赖于该属性的 Reactive Effect。

Vue3的优化：
（1）清除无效依赖：在每次依赖收集之前，Vue 会清除之前的依赖，确保依赖关系是最新的。
（2）惰性依赖收集：只有在 Reactive Effect 执行时才会收集依赖，避免了不必要的依赖关联。

4. v-if和v-show的区别

  • v-show通过CSS的display属性控制元素的显示与隐藏。即使条件为false，元素依然会被渲染到DOM中，只是不可见。v-if则是通过条件渲染，当条件为false时，元素不会被渲染到DOM中，当条件变为true时，元素才会被渲染。
  • v-if有更高的切换消耗，因为涉及到DOM的创建和销毁。v-show则有更高的初始渲染消耗，因为无论条件如何，元素始终会被渲染。

5. 什么是keep-alive

keep-alive是vue中的内置组件，能在组件切换过程中将状态保留在内存中，防止重复渲染DOM

6. history和hash路由模式

  • URL结构：hash包含#
  • SEO：history路由模式对SEO更友好（完整的URL路径可以被搜索引擎抓取，‘#’后面的内容可能会被搜索引擎忽略
  • 兼容性：hash路由模式兼容所有浏览器，history模式部分低级浏览器不兼容
  • 实现原理：history模式基于pushstate和replacestate实现，hash模式通过监听浏览器的onhashchange（）事件查找对应的路由规则。
  • 刷新：hash（#及后面的内容）的变化不会导致浏览器向服务器发送请求，history刷新会重新向服务器发送请求，需要在服务端做处理，如果没有匹配到资源就返回同一个html页面。（注意：两种模式单纯url变化都不会立即向服务器发送请求，history模式下需要刷新

15. watch和computed的区别

watch：用来监听某个数据的变化，并在数据变化的时候执行自定义回调函数

computed：是vue提供的一种计算属性，依赖于其他响应式数据，并在其依赖的数据发送变化的时候自动更新

16. ref和reactive的区别

  • 数据类型：ref支持基本数据类型和对象类型，reactive仅支持对象类型
  • 访问方式：ref必须通过.value操作，reactive可以直接属性
  • 重新赋值：ref允许整个替换（实际操作的是 .value，reactive不能直接替换整个对象

17. vue和react的区别

  • dom的更新策略不同：react 会自顶向下全diff。vue会跟踪每一个组件的依赖关系,不需要重新渲染整个组件树。
  • 数据驱动视图：vue使用mvvm框架实现数据驱动视图更新，react是通过setState来引发一次组件的更新过程从而实现页面的重新渲染（react需要手动触发）。
  • 组件化的差异：react使用jsx+inline开发组件，vue推荐单文件的组件格式

计算机网络

🔥 计算机网络面经

1. OSI模型

应用层：为应用程序或者用户请求提供各种请求服务（HTTP、DNS、SMTP)

传输层：端到端，可靠传输服务(TCP、UDP)

网络层：主机到主机，路由选择和转发（IP、ICMP）

链路层：在相邻节点之间传输数据帧，提供错误检测和纠正功能。（以太网/PPP/ARP）

物理层：物理介质上的传输

2. HTTP不同版本的区别

HTTP1.0: 短连接（每一次响应请求都要重新建立连接）+ expires字段

HTTP1.1: 持久连接（Keep-Alive，多个响应和请求可以使用同一个TCP连接）+ 管道化（允许客户端在同一个TCP连接上连续发送多个请求，而无需等待上一个请求的响应返回）+ cache-control字段

HTTP2.0: 二进制分帧（将请求和响应分解为更小的二进制帧）+ 多路复用（并行，解决了队头堵塞【响应需要按顺序返回】的问题，使用帧+流ID标识来进行乱序处理）+ 头部压缩

HTTP3.0: QUIC协议（基于udp）

3. HTTP头部字段

  • 请求头

    1. Accept：指定客户端能够接受的内容类型
    2. Authorization：用于验证身份的头部
    3. Cookie：发送给服务器的cookie信息
    4. Host：指定请求的目标主机和端口号
    5. UserAgent：标识发出请求的用户代理（通常是浏览器）例如：User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36

  • 响应头

    1. Content-Type：响应体的内容类型
    2. Content-Length：响应体的长度
    3. Cache-Control：指定缓存策略
    4. Set-Cookie：用于在客户端设置cookie
    5. Date：响应被发送的时间
    6. Server：服务器的软件信息

  • 通用头/其他头部

    1. Connection：指定是否保持连接或者关闭连接，例如：Connection：keep-alive
    2. Location：指定重定向的目标的URL
    3. ETag：资源的标识符，用于缓存控制

4. HTTPS

在HTTP的基础上加入了SSL/TLS协议的一种安全通信协议。它通过对传输的数据进行加密，确保数据在客户端和服务器之间的传输过程中不会被窃取或篡改。

  • 对称加密：加密和解密都是使用同一个密钥，常见的对称加密算法有 DES、3DES 和 AES 等

  • 非对称加密：加密和解密需要使用两个不同的密钥，公钥（public key）和私钥（private key）。公钥与私钥是一对，如果用公钥对数据进行加密，只有用对应的私钥才能解密；如果用私钥对数据进行加密，那么只有用对应的公钥才能解密。非对称加密算法实现机密信息交换的基本过程是：甲方生成一对密钥并将其中的一把作为公钥对外公开；得到该公钥的乙方使用公钥对机密信息进行加密后再发送给甲方；甲方再用自己保存的私钥对加密后的信息进行解密。常用的非对称加密算法是 RSA 算法。

5. localstorage、cookie、sessionstorage的区别

  • 存储时效：localstorage永久存储（除非手动清除），sessionStorage会话级（关闭浏览器/标签页失效），cookie可设置过期时间（默认为会话级）
  • 存储大小：localStorage 5-10MB（不同浏览器有差异），sessionStorage 同localStorage，cookie 单个约4KB
  • 与服务器通信：localStorage 不自动参与，sessionStorage 不自动参与，cookie 每次HTTP请求自动携带
  • 访问范围：localStorage 同源窗口共享，sessionStorage 仅当前标签页有效（同源也不共享），cookie默认同源窗口共享（可设置domain/path限制共享范围）
  • 典型应用：localStorage 适合长期保存用户偏好，sessionStorage 适合临时存储表单数据，cookie 适合身份认证和会话跟踪

6. 常见的攻击手段

XSS(跨站脚本攻击): 在html页面注入恶意代码，执行恶意脚本。防范手段：（1）输入过滤（2）编码转换

CSRF(跨站请求伪造）：伪造受信任用户的身份进行资源请求。防范手段：（1）token（ 与 Cookie 自动附带 到请求不同，Token 需要通过 JavaScript 手动添加到 Authorization 头， 浏览器的同源策略禁止跨域 JS 访问其他域的 Token ）

7. 常见的状态码

  • 1–：接受的请求正在处理
  • 2–：成功（200表示请求成功）
  • 3–：缓存重定向（300表示资源永久重定向、301表示资源暂时重定向、304表示资源未重定向）
  • 4–：客户端错误（403表示无权限访问、404表示资源不存在）
  • 5–：服务器错误（500表示服务器内部错误）

8. DNS缓存

强缓存：指的是浏览器可以直接使用缓存的资源，而无需向服务器发送请求验证资源的有效性。当资源的缓存未过期时，浏览器会直接从本地缓存读取资源。expires：绝对时间，cache-control：相对时间（max-age）

协商缓存：在缓存过期后，浏览器发送请求到服务器，由服务器根据请求头的特定字段判断是否更新。如果资源未更新，服务器返回 304 Not Modified，浏览器继续使用本地缓存。last-modified：资源最后修改的时间；if-modified-since: 上次携带的last-modified的值；E-tag：服务器为资源生成的唯一标识（一般是哈希值）；if-none-match：浏览器在请求中携带上次的 ETag 值

9. 三次握手和四次挥手

  • 三次握手：目的是确认双方具备收发能力以及同步初始序列号

    1. 客户端发送SYN请求
    2. 服务器发送SYN-ACK确认
    3. 收到SYN-ACK报文后，会回应ACK确认

    为什么不能二次握手：无法确认客户端的接收能力是否正常，防止旧的重复连接造成混乱。

    为什么不需要四次握手或以上：避免资源的浪费

  • 四次挥手：目的是确保双方数据完全传输完毕并安全关闭连接

    1. 主动方发送FIN终止请求
    2. 被动方回应ACK确认收到
    3. 被动方发送FIN终止请求
    4. 主动方回应ACK确认收到

    为什么不能少于四次挥手：被动方收到FIN后可能还有数据要发送，需要等待数据传输完毕才能发送自己的FIN。且主动方需要保证数据完整接收。

🧱 一、JavaScript 基础

1. 数据类型

1.1 基础数据类型和引用数据类型

基础数据类型 ：string,number,bigInt,undefined,null,symbol,boolean

引用数据类型 : object、Array、RegExp、Function，Date、特殊的基本包装类型（String、Number、Boolean）、单体内置对象（Math、Global）

区别：（1）基本数据类型的值存储在栈内存中（固定大小），引用数据类型的值存在堆中，引用指针存储在栈中（2）基本数据类型的值不可变（本质：每次”修改”都是创建新值，变量指向新地址），引用数据类型的值可变（本质：修改的是堆内存中的数据，引用地址不变）（3）进行比较时，基本数据类型比较值，引用数据比较引用是否指向同一对象。

1.2 数据判断的几种方法

  • typeof：只能用于判断基本数据类型和简单引用类型（数组），存在局限：typeof 数组 = Object , typeof null = Object 且 typeof 对象不能得到具体的类型。

  • instanceof: 不能用于判断基本数据类型，用于判断复杂的引用数据类型。a instanceof A 是通过判断A(构造函数)的原型（prototype）是否位于对象a的隐式原型链（proto）上（用于判断一个实例是否属于某个数据类）

  • constructor：可以用于判断基本数据类型和复杂的引用数据类型。a.constructor = A 是沿着a的隐式原型链上查找constructor属性，检查有与A（构造函数）相等的。（用于判断一个实例的constructor属性是否符合预估类）缺点在于constructor属性容易被修改，导致判断有误。

  • Object.prototype.toString().call()：基本和引用数据类型皆可判断，但自定义类只会显示[Object Object],内置类可以显示具体情况。

💡 TIP
typeof会将null判断为Object，原因是typeof会将所有传进去的值都转成二进制，而js制定的规则会认为二进制前三个值为0的是对象，遵循其他语言的原则js将null定为一长串的0，所以会被误判为对象

💡 TIP
instanceof和constructor判断基本数据类型的区别在于：基本数据类型不是对象，所以没有原型链，而constructor会将其隐式包装为对象

💡 TIP
JavaScript 引擎在创建每个对象时，都会给它分配一个 [[Class]] 内部属性，用于标识对象的类型。这个属性是 不可变的，无法通过 JavaScript 代码直接修改，因此 Object.prototype.toString.call() 能稳定返回正确的类型。

1.3 null和undefined的区别

null是开发者主动赋值的空值，undefined是js的默认缺省（表示未定义或者未初始化）

1.4 map和object的区别

  • map的键值可以是任何类型，而object的键值只能是string或者symbol
  • map的键值是按插入顺序排列的，object不一定是。

2. 变量声明和作用域

2.1 var、const和let的区别

  • 变量提升：var存在变量提升，而const和let无。（变量提升是指声明提升到作用域的顶部，但是初始化不提升）
  • 作用域：var的作用域为函数作用域或全局作用域，let和const的作用域为块级作用域。
  • 重复声明：var可以重复声明，let和const不可以重复声明
  • 重新赋值：var和let可以重新赋值，const声明基本数据类型时不可重新赋值，声明对象/数组不可以修改引用，但可以修改对象/数组的属性和元素。

2.2 作用域

2.3 闭包

闭包是指一个函数有权访问另一个函数作用域的变量。闭包就是函数的执行上下文被保留，即使外部函数已经执行完毕，内部函数仍然可以访问外部函数的变量。

  • 闭包可以创建私有变量，避免全局污染：
  • 回调函数
  • 函数工厂（其中经典的有函数柯里化）
  • 防抖/节流函数

💡 TIP
闭包导致变量不会被垃圾回收机制回收，造成内存消耗闭包中的变量不会被销毁，原因在于闭包保留了对变量的引用，形成了一个作用域链。只要闭包存在，它所引用的变量就会一直存在于内存中，不会被垃圾回收机制回收。

2.4 this函数的指向的几种方式

  • 使用call、bind、apply指定this指向
  • 通过new将this固定在实例上
  • 以函数的形式调用，this的指向是window
  • 以类方法的形式调用，this的指向是调用这个方法的对象
  • 全局作用域下，this的指向是window

2.5 call、apply、bind的区别

  • call：参数逐个列出，立即执行 func.call(thisArg, arg1, arg2, …)
  • apply：参数以数组形式传入，立即执行，func.apply(thisArg, [argsArray])
  • bind：参数逐个列出且可以部分应用参数（柯里化），不会立即执行，需要手动调用

🚀 二、核心概念

1. 异步

1.1 事件循环

js是单线程的，为了避免阻塞，会将同步和异步任务分到不同的执行环境，同步的进入主线程，即主执行栈，异步的进入任务队列。每次执行事件时会询问主执行栈情况，当主线程内的任务执行完毕后，会去任务队列读取对应的任务，推入主线程执行。上述过程不断重复就是事件循环。

1.2 宏任务和微任务

宏任务和微任务都是异步任务，微任务是更高优先级的任务队列，提供了一种事件插队的机制，务队列分为微任务队列和宏任务队列，每次执行完同步任务后，会先询问微任务队列情况，先执行微任务队列再执行宏任务队列。（宏任务是由宿主（浏览器）起的；常见的宏任务：settimeout、setinterval；常见的微任务：promise.then(),promise.catch()、nexttick（）（要先于promise）

1.3 同步和异步的区别

同步：按照顺序执行任务，必须等待前一个任务完成
异步：发起任务后继续执行后续逻辑，通过回调或者事件通知结果

1.4 promise

  • 定义：promise是js中用于处理异步操作的对象，具有三种状态，pending（进行中）、fulfilled（操作完成）、rejected（操作失败）；Promise的构造函数接受一个函数作为参数，这个函数有两个参数：resolve和reject。resolve用于在异步操作成功时调用，将结果传递出去，修改promise对象的状态以及执行回调函数；reject则是在操作失败后调用，将错误信息传递出去，修改promise对象的状态以及执行相应的处理函数

  • Promise的链式调用：Promise的链式调用是通过.then()实现的，每次调用.then()都会返回新的promise，这样就可以继续调用.then()或者.catch()。如果promise被reject或者在.then()中抛出错误，后续会跳过.then(),直接执行最近的catch来处理错误。使用链式调用可以很好解决回调地狱问题，可以使用try-catch统一捕获整个链中的错误，不需要在每个回调函数中单独处理错误。

1.5 async/await

async：async用于声明一个异步函数，会将返回值包装成一个promise对象，如果函数返回一个值，promise会以该值resolve；如果函数抛出异常，会以该异常reject。

await：await只能在async内执行，用于暂停async的执行，直到等待的promise被解决或者拒绝

优点：（1）避免回调地狱和promise链式调用，在语义和代码风格上更易理解
（2）可以使用try-catch，统一处理错误

🌐 四、浏览器知识

1. 事件冒泡、事件委托

事件冒泡：指一个事件在子元素上触发时，它会向上冒泡到父元素、祖父元素，直到文档的根元素。
事件委托：通过在父元素上监听事件，可以根据事件的target属性来判断事件实际发生在哪个子元素上，从而执行相应的处理逻辑

优点：

  • 减少内存消耗，只需要绑定一个事件处理器，而不是为每个子元素都绑定一个。
  • 对于动态添加的子元素，无需重新绑定事件处理器
  • 简化代码，易于维护

2. 事件传播的三个阶段

（1）捕获阶段：先由文档的根节点往内部触发对象，从外向内直到捕获到目标对象
（2）目标阶段：事件到达目标元素，触发相应的处理事件
（3）冒泡阶段

3. 阻止冒泡的原因

（1）避免多重触发，父元素和子元素绑定相同类型的事件处理器会触发多次，造成逻辑错误
（2）阻止事件冒泡可以减少不必要的事件处理器执行，从而提高性能
