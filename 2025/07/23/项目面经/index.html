  • webworker：

    Web Worker 的原理是通过浏览器提供的多线程机制，在主线程之外创建后台线程运行 JS 脚本。它通过 postMessage 实现和主线程的数据通信，避免主线程卡顿。

    它最大的优点是：可以在不阻塞 UI 的情况下执行大量计算任务，提升性能体验，非常适合数据处理、图像处理、加密等场景。

    不过，它也有一些限制，比如不能访问 DOM、数据传递是复制而非共享、有内存开销等。

1.3 OAuth2鉴权是如何实现的

  1. 用户 → 客户端应用
    用户授权请求

  2. 客户端应用 → 授权服务器（附带 redirect_uri）
    跳转到授权服务器

  3. 授权服务器 → 用户
    用户登录并同意授权

  4. 授权服务器 → 客户端应用（通过 redirect_uri 携带授权码重定向）
    返回授权码（code）

  5. 客户端应用 → 令牌端点
    发送授权码请求令牌（带 code、client_id、client_secret、redirect_uri）

  6. 令牌端点 → 客户端应用
    返回访问令牌（access_token）

  7. 客户端应用 → 受保护资源
    使用 access_token 访问资源

  8. 受保护资源 → 客户端应用
    返回资源数据

  9. 客户端应用 → 令牌刷新端点
    令牌过期，使用 refresh_token 刷新令牌

  10. 令牌刷新端点 → 客户端应用
    返回新的访问令牌

  11. 客户端应用
    访问令牌刷新成功
